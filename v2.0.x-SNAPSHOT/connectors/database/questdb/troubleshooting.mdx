## QuestDB-Specific Issues

### Connection Refused on Default Port

QuestDB listens on port **8812** for the PostgreSQL wire protocol - not the standard PostgreSQL port 5432. Ensure your `hostPort` is set to `<host>:8812` and that the port is reachable from the ingestion host.

### Schema Filter Not Supported

QuestDB operates as a single-database (`qdb`), single-schema (`public`) system. Schema filtering is not applicable - all tables reside in the `public` schema and will be ingested automatically. You can use `tableFilterPattern` to restrict which tables are included.
