## Requirements

The Google Cloud service account used for ingestion needs the following IAM permissions:

### Metadata Ingestion

| Permission | Purpose |
|---|---|
| `pubsub.topics.list` | List topics in the project |
| `pubsub.subscriptions.list` | List subscriptions (for dead letter detection and subscription metadata) |
| `pubsub.subscriptions.get` | Read individual subscription details |

### Schema Registry (when `schemaRegistryEnabled` is `true`)

| Permission | Purpose |
|---|---|
| `pubsub.schemas.list` | List schemas in the Schema Registry |
| `pubsub.schemas.get` | Read schema definitions (Avro, Protocol Buffer) |

The built-in GCP role **`roles/pubsub.viewer`** grants all of the above permissions and is the recommended role for OpenMetadata ingestion.

### Python Requirements

<PythonRequirements />

To run the Google Pub/Sub ingestion, you will need to install:

```bash
pip3 install "openmetadata-ingestion[pubsub]"
```

Configure the source type and service name for your Google Pub/Sub connector.

<ContentSection id={2} title="GCP Credentials" lines="7-17">

**gcpConfig**: GCP service account credentials for authenticating with Pub/Sub.

Provide a service account key JSON with the following fields:
- **type**: Always `service_account`
- **projectId**: Your GCP project ID
- **privateKeyId**: The ID of the private key
- **privateKey**: The RSA private key (include the full `-----BEGIN/END RSA PRIVATE KEY-----` block)
- **clientEmail**: The service account email (e.g., `ingestion@project.iam.gserviceaccount.com`)
- **clientId**: The numeric client ID of the service account

For more information, see [Creating and managing service account keys](https://cloud.google.com/iam/docs/creating-managing-service-account-keys).

</ContentSection>

<ContentSection id={3} title="Project ID" lines="18">

**projectId** (optional): GCP Project ID where Pub/Sub topics are located. If not specified, the project ID is automatically read from the service account credentials.

</ContentSection>

<ContentSection id={4} title="Host and Port / Use Emulator" lines="19-20">

**hostPort** (optional): Pub/Sub API endpoint. Defaults to `pubsub.googleapis.com`.

**useEmulator** (optional): Set to `true` to connect to a local [Pub/Sub emulator](https://cloud.google.com/pubsub/docs/emulator) instead of the production service. When enabled, `hostPort` must point to the emulator address (e.g., `localhost:8085`).

</ContentSection>

<ContentSection id={5} title="Schema Registry" lines="21">

**schemaRegistryEnabled** (optional, default: `true`): Fetch topic schemas from the Pub/Sub Schema Registry. Supports **Avro** and **Protocol Buffer** schema types. Set to `false` if your project does not use the Schema Registry.

</ContentSection>

<ContentSection id={6} title="Include Subscriptions" lines="22">

**includeSubscriptions** (optional, default: `true`): Include subscription metadata for each topic. When enabled, subscription names, acknowledgment deadlines, retention durations, push endpoints, dead letter policies, and BigQuery export configurations are captured in OpenMetadata.

When a subscription has a **BigQuery export configuration**, OpenMetadata automatically extracts lineage from the Pub/Sub topic to the target BigQuery table.

</ContentSection>

<ContentSection id={7} title="Include Dead Letter Topics" lines="23">

**includeDeadLetterTopics** (optional, default: `false`): Include dead letter topics in metadata extraction. By default, dead letter topics are detected via subscription policies and excluded to keep the topic list focused on primary business topics.

</ContentSection>

<ContentSection id={8} title="Topic Filter Pattern" lines="24-28">

**topicFilterPattern**: Regex pattern to selectively include or exclude topics by name.

- **includes**: List of regex patterns — only matching topics are ingested
- **excludes**: List of regex patterns — matching topics are skipped

Example: to exclude internal topics prefixed with `_`, add `"^_.*"` to `excludes`.

</ContentSection>

</ContentPanel>

<CodePanel fileName="pubsub_config.yaml">

```yaml
source:
  type: pubsub
  serviceName: gcp_pubsub
  serviceConnection:
    config:
      type: PubSub
      gcpConfig:
        gcpConfig:
          type: service_account
          projectId: <gcp-project-id>
          privateKeyId: <private-key-id>
          privateKey: |
            -----BEGIN RSA PRIVATE KEY-----
            <your-private-key>
            -----END RSA PRIVATE KEY-----
          clientEmail: ingestion@<gcp-project-id>.iam.gserviceaccount.com
          clientId: "<client-id>"
      # projectId: <gcp-project-id>          # Optional: override project from credentials
      # hostPort: pubsub.googleapis.com       # Default; set to localhost:8085 for emulator
      # useEmulator: false                    # Set to true for local Pub/Sub emulator
      # schemaRegistryEnabled: true           # Fetch schemas from Schema Registry
      # includeSubscriptions: true            # Capture subscription metadata
      # includeDeadLetterTopics: false        # Include dead letter topics
      # topicFilterPattern:
      #   includes:
      #     - ".*"
      #   excludes:
      #     - "^_.*"
      # supportsMetadataExtraction: true
```
