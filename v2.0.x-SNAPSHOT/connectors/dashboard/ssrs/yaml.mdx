## Requirements

The SSRS connector uses the [SSRS REST API](https://docs.microsoft.com/en-us/sql/reporting-services/developer/rest-api) (v2.0) and authenticates via **Windows Authentication (NTLM)**. The account used must have at minimum **Browser** role on the SSRS Report Server to list folders and reports.

<Info>
The SSRS REST API (v2.0) requires **SQL Server Reporting Services 2017 or later**. Older SSRS versions using SOAP-based web services are not supported.
</Info>

To grant the required role:
1. Open the **SSRS Web Portal**.
2. Navigate to **Site Settings** → **Security** → **New Role Assignment**.
3. Add the service account and assign the **Browser** role.

<ContentSection id={2} title="Host and Port" lines="7">

**hostPort**: The base URL of your SSRS Report Server, e.g., `https://your-ssrs-server/reportserver`. The connector appends `/api/v2.0` to this URL when calling the REST API.

</ContentSection>

<ContentSection id={3} title="Username" lines="8">

**username**: The Windows account username for NTLM authentication. Use `DOMAIN\username` format for domain accounts.

</ContentSection>

<ContentSection id={4} title="Password" lines="9">

**password**: Password for the Windows account used for NTLM authentication.

</ContentSection>

<ContentSection id={5} title="Verify SSL" lines="10">

**verifySSL** *(optional)*: Controls SSL certificate verification — `no-ssl` (default), `ignore`, or `validate`.

</ContentSection>

<ContentSection id={6} title="SSL Configuration" lines="11-12">

**sslConfig** *(optional)*: Required when `verifySSL` is `validate`. Provide the CA certificate to verify the SSRS server's SSL certificate.

</ContentSection>

<ContentSection id={7} title="Filter Patterns" lines="13-24">

**dashboardFilterPattern**, **chartFilterPattern**, **projectFilterPattern** *(optional)*: Regex patterns to include or exclude SSRS reports, charts, and folders from ingestion.

</ContentSection>

<CodePanel fileName="ssrs_config.yaml">

```yaml
source:
  type: ssrs
  serviceName: <service_name>
  serviceConnection:
    config:
      type: Ssrs
      hostPort: <https://your-ssrs-server/reportserver>
      username: <username>
      password: <password>
      # verifySSL: no-ssl
      # sslConfig:
      #   caCertificate: <path-to-ca-cert>
      # dashboardFilterPattern:
      #   includes:
      #     - ".*"
      #   excludes: []
      # chartFilterPattern:
      #   includes:
      #     - ".*"
      #   excludes: []
      # projectFilterPattern:
      #   includes:
      #     - ".*"
      #   excludes: []
```
