# Codebase Deep Dives

These guides explain how key OpenMetadata modules work under the hood. They are written for contributors who want to understand the call chains, class hierarchies, and design patterns before diving into the code.

- [Data Quality](/v1.13.x/developers/contribute/codebase-deep-dives/data-quality) — Test definitions, validators, execution pipeline, and how to add new tests
- [Profiler](/v1.13.x/developers/contribute/codebase-deep-dives/profiler) — Metrics system, computation engine, threading model, and how to add new metrics
- [Sampler](/v1.13.x/developers/contribute/codebase-deep-dives/sampler) — Sampling strategies, database-specific implementations, and integration with profiler and data quality
- [Metadata Ingestion](/v1.13.x/developers/contribute/codebase-deep-dives/metadata-ingestion) — Topology-based execution, source hierarchy, and how connectors extract metadata
- [Usage Ingestion](/v1.13.x/developers/contribute/codebase-deep-dives/usage-ingestion) — Query log collection, SQL parsing, aggregation, and publishing
- [Lineage Ingestion](/v1.13.x/developers/contribute/codebase-deep-dives/lineage-ingestion) — SQL parsing, graph analysis, column-level lineage, and cross-database lineage
