## SSRS-Specific Issues

### NTLM Authentication Failure

If the connector fails with a `401 Unauthorized` error, verify that:
- The username uses the correct format (`DOMAIN\username` for domain accounts)
- The account has at least the **Browser** role on the SSRS Report Server
- NTLM authentication is enabled on the Report Server (not only Kerberos)

### Cannot Connect to SSRS REST API

Ensure the SSRS REST API is enabled. The connector uses `/api/v2.0/` endpoints — this requires **SSRS 2017 or later**. Older versions using SOAP-based web services are not supported by this connector.

### SSL Certificate Errors

If you see SSL verification errors, either:
- Set `verifySSL: ignore` to skip validation (not recommended for production)
- Set `verifySSL: validate` and provide the correct CA certificate path in `sslConfig`

### Reports Not Appearing

If reports are missing after ingestion:
- Confirm the service account has **Browser** role access on the folder containing those reports
- Check whether `dashboardFilterPattern` or `projectFilterPattern` is inadvertently excluding them
- Hidden reports are automatically skipped by the connector
